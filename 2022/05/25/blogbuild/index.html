记录下如何搭建的这个博客

发表于 2022-05-25 更新于 2022-05-21 分类于 博客搭建

主要步骤

  1. 安装node 和 git
  2. 安装hexo并配置
  3. 注册github并且创建和github账户同名的.github.io仓库
  4. 生成新的SSH密钥
  5. 本地hexo文件更新到github
  6. 创建博客内容并上传
  7. 上传图片路径解析无法显示的问题及解决方案
  8. 配置定制化界面

安装node和git

  • node 和 git 的安装直接在百度上搜官网下载即可， node下载安装完成后，自带npm

  • 可以通过node -v 和 git version 查看版本信息 判断是否安装成功

  • git 安装成功之后，任意文件夹下右键会有git bash here选项

  • git下载完成之后需要做以下配置：

    • git config –global user.name xxx // 配置用户名
    • git config –global user.email xxx // 配置邮箱
    • git config –global http.postBuffer 524288000 // 防止后面操作可能出现的报错：SSL_ERROR_SYSCALL,errno 10054

  • 配置完成之后，可以通过 git config –list 查看

安装hexo并配置

&emsp;&emsp; hexo 也是npm管理的一个package，可以直接通过npm 下载

  • 切换到准备好的文件夹下，右键git bash here 打开bash窗口，执行 npm i -g hexo-cli 下载hexo脚手架

  • 通过hexo-cli 脚手架初始化： hexo init

  • npm install 或 npm i 下载依赖

  • 下载hexo部署工具 npm i hexo-deployer-git –save

  • 打开hexo官网 找到 左侧的命令 ,执行hexo g ,然后再执行 hexo server,到这一步本地可以查看博客页面了,注意按照bash窗口提示 ctrl+点击是不会带端口号的，加上端口号即可

注册github并且创建和github账户同名的.github.io仓库

  • 打开github官网，比较慢的情况下按下图配置会稍微好点，先注册登陆

点击头像选择 your repositories ,新建一个github账号同名的仓库 账号名.github.io作为仓库名称，后续简称博客仓库

生成新的SSH密钥

  • 首先检查之前有没有设置过使用ssh登陆连接github，可以通过命令行 cd ~/.ssh 查看是否有做过配置

  • 如果没有创建新的ssh秘钥并添加到github

  • 如图中指示，创建新的ssh秘钥：ssh-keygen -t rsa -b 4096 -C “your_email@example.com“ 此处的邮箱是git 配置的 连续回车就好，如果之前生成过，rewrite：y 覆盖

  • 直接在这个窗口中选中右键复制的格式是不正确的。。执行脚本 clip < ~/.ssh/id_rsa.pub

  • 之前的页面测试

本地hexo文件更新到github

  • 克隆博客仓库到本地，可能需要再次配置 http.postBuffer

  • 修改配置，后续页面个性化配置也需要改这个文件

  • 部署 hexo g, 然后 hexo d 可以合并为 hexo g -d 或 hexo d -g 第一次会有邮箱密码验证

  • 访问用户用.github.io

创建博客内容并上传 每次修改执行 2 3

  1. hexo new xxx 会在source文件夹下新建一个.md文件 ，编写内容
  2. hexo clean
  3. hexo g -d 查看更新的内容

上传图片路径解析无法显示的问题及解决方案

  • 本地上传的图片在上传到github之后，由于路径解析的问题，无法加载图片
  • 下载hexo-asset-image 插件 可以在npm官网按这个名称搜索，找一个最近更新的 然后修改源代码，多打包两次看下解析生成路径，直到正确为止

配置定制化界面

  • 网络允许的情况下可以使用hexo官网推荐的主题。或者主流：next 配置文档 或者官网配置
  • clone主题仓库到themes文件夹
  • 按照文档进行个性化配置即可
